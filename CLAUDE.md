## 1. 핵심 원칙
- **기본 모델: Sonnet** — 작업의 80%는 Sonnet으로 처리. 토큰 비용 60% 절감.
- **Opus 전환 트리거**: 아래 조건 중 하나라도 해당하면 사용자에게 `/model opus` 전환을 제안할 것.
- **컨텍스트는 자산이다** — 불필요한 정보가 쌓이면 성능이 저하된다(Context Rot). 적극적으로 관리할 것.

---

## 2. 모델 전환 전략

### Sonnet 유지 (기본)
일반 코딩, 파일 수정, 간단한 버그 수정, 리팩토링, 테스트 작성, 문서 작성, 단순 질의응답

### Opus 전환 제안 시점
다음 상황에서 **반드시** 사용자에게 알릴 것:
- 시스템 아키텍처 설계 또는 대규모 구조 변경
- 3회 이상 동일 버그 해결 실패 (디버깅 루프 탈출 불가)
- 복잡한 멀티파일 의존성 분석이 필요한 경우
- 보안 취약점 분석 또는 코드 리뷰
- 새로운 프로젝트 초기 설계 (scaffolding)
- 성능 최적화에서 병목 원인 추론이 필요한 경우

**전환 제안 형식:**
```
⚡ 이 작업은 복잡도가 높아 Opus가 더 적합합니다.
전환: /model opus
완료 후 복귀: /model sonnet
```

### Opus → Sonnet 복귀 알림
Opus로 전환한 작업이 완료되면 **즉시** 사용자에게 Sonnet 복귀를 제안할 것.

---

## 3. 토큰 절약 운영 규칙

### 응답 스타일
- 불필요한 서론, 요약 반복, 과도한 설명 금지
- 코드 변경 시 전체 파일 출력 대신 diff/변경 부분만 표시
- 확인 질문은 한 번에 모아서 할 것 (왕복 최소화)
- 이미 컨텍스트에 있는 파일 내용을 재출력하지 말 것

### 컨텍스트 다이어트
- **작업 전환 시**: 무관한 이전 작업의 컨텍스트가 남아있으면 사용자에게 `/clear` 제안
- **마일스톤 도달 시**: 기능 하나 완성, 버그 하나 해결 등 단위 작업 완료 시 `/compact` 제안
- **컨텍스트 50% 이상 소모 감지 시**: 즉시 `/compact` 제안

**제안 형식:**
```
📦 현재 작업이 완료되었습니다. 컨텍스트 최적화를 위해 /compact를 권장합니다.
```
```
🔄 새로운 작업으로 전환합니다. /clear로 컨텍스트를 리셋하면 성능이 유지됩니다.
```

## 4. MCP 서버 관리
- 프로젝트당 MCP 서버 **10개 이하** 유지
- 활성 도구 **80개 이하** 유지
- 사용하지 않는 MCP 서버는 `disabledMcpServers`로 비활성화할 것
- MCP 도구 설명이 컨텍스트 윈도우를 20만→7만 토큰까지 축소시킬 수 있음을 인지할 것

## 5. 연속 학습 시스템 (Continuous Learning)

### 본능(Instinct) 기록
세션 중 다음 패턴을 감지하면 `~/.claude/skills/learned/`에 기록할 것:
- **사용자 교정**: 사용자가 코드를 수정하거나 접근법을 바꾼 경우
- **에러 해결**: 특정 에러의 해결 패턴이 반복되는 경우
- **워크어라운드**: 프레임워크/라이브러리 특유의 우회 방법
- **프로젝트 컨벤션**: 네이밍, 디렉토리 구조, import 순서 등

### 본능 → 스킬 승격
동일 도메인의 본능이 **3개 이상** 축적되면:
1. 관련 본능들을 클러스터링
2. 재사용 가능한 스킬로 통합 제안
3. 사용자 승인 후 `~/.claude/skills/` 에 스킬 파일 생성

### 본능 형식
```yaml
---
id: [패턴-식별자]
trigger: "[트리거 조건]"
confidence: 0.5  # 초기값, 반복 확인 시 상승
domain: "[code-style|testing|debugging|project-specific]"
---
# [패턴 이름]
## Action
[수행할 행동]
## Evidence
[관찰 근거]
```

## 8. 금지 사항
- 동일한 내용을 다른 말로 반복 설명하지 말 것
- "이해했습니다", "물론이죠" 등 무의미한 확인 응답 최소화
- 전체 파일 내용을 불필요하게 출력하지 말 것
- 사용자가 요청하지 않은 추가 설명/주석을 과도하게 붙이지 말 것
- 추측으로 코드를 작성하지 말 것 — 불확실하면 먼저 확인할 것

## 9. settings.json 권장 설정 (별도 적용)

```jsonc
// ~/.claude/settings.json
{
  "model": "sonnet",
  "MAXTHINKINGTOKENS": "10000",
  "CLAUDECODESUBAGENT_MODEL": "haiku"
}
```

| 설정 | 효과 |
|------|------|
| `model: sonnet` | 기본 모델 비용 60% 절감 |
| `MAXTHINKINGTOKENS: 10000` | 생각 토큰 제한으로 숨은 비용 70% 절감 (기본 31,999) |
| `CLAUDECODESUBAGENT_MODEL: haiku` | 서브에이전트 비용 80% 절감 |
